## 3. Current best-practice config schema (v1.18.1)

### 3.1 Top-level `opencode.json` keys (relevant subset)

| Key | Type | Notes |
|---|---|---|
| `$schema` | string | `https://opencode.ai/config.json` |
| `model` | `provider/model-id` | Default for the primary agent. |
| `small_model` | `provider/model-id` | Used for titles, summaries. Defaults to a cheaper model if available. |
| `instructions` | string[] | Glob-supported list of instruction files (project-relative). |
| `provider` | object | Per-provider config: `options`, `models`, `blacklist`, `whitelist`, `npm`, `name`, `baseURL`. |
| `permission` | object or string | Default-deny by default; `"allow" | "ask" | "deny"` per tool. Wildcards supported. |
| `mcp` | object | MCP servers: `type: "local" | "remote"`, `command[]`, `url`, `headers`, `oauth`, `timeout`, `enabled`, `environment`. |
| `agent` | object | Custom agents (`primary` or `subagent`). Markdown form goes in `.opencode/agents/<name>.md`. |
| `command` | object | Custom commands. Markdown form goes in `.opencode/commands/<name>.md`. |
| `tools` | object | **DEPRECATED since v1.1.1** — use `permission` instead. Still works for back-compat. |
| `lsp` | object/boolean | LSP server enable/config. |
| `formatter` | object/boolean | Formatter enable/config. |
| `snapshot` | boolean | Internal change tracking. Default `true`. Disable for huge repos. |
| `share` | `"manual" | "auto" | "disabled"` | Sharing mode. Default `manual`. |
| `autoupdate` | boolean \| `"notify"` | Default `true`. |
| `default_agent` | string | Which primary agent is default. |
| `disabled_providers` | string[] | Blacklist. |
| `enabled_providers` | string[] | Whitelist. Wins over `disabled_providers` only via per-key priority rule. |
| `experimental` | object | `policies` etc. — unstable. |
| `server` | object | `port`, `hostname`, `mdns`, `cors`. |
| `shell` | string | Force a specific shell. |
| `compaction` | object | `auto`, `prune`, `reserved`. |
| `watcher` | object | `ignore: string[]` — glob patterns. |
| `plugin` | string[] | npm plugin names. |
| `attachment` | object | Image limits. |
| `theme`, `keybinds`, `tui` | — | **DEPRECATED in `opencode.json`**; use `tui.json`. Auto-migrated. |

### 3.2 Directory layout
OpenCode walks up from the cwd to the git worktree. Project-local config lives in:
```
.opencode/
├── agents/      # *.md → custom subagents
├── commands/    # *.md → custom /commands
├── skills/      # <name>/SKILL.md → on-demand skills
├── plugins/     # local plugins
├── tools/       # custom tools
└── themes/      # custom themes
```

Skill/agent/command definitions are also discovered at Claude-compatible paths (`.claude/skills/`, `~/.claude/skills/`, `.agents/skills/`, `~/.agents/skills/`) — useful for retrocompat.

### 3.3 Frontmatter rules (skills, agents, commands)

**SKILL.md frontmatter (only these fields are recognized):**
- `name` — required. 1–64 chars, lowercase alphanumeric + single hyphens, must match directory name.
- `description` — required. 1–1024 chars. Front-load trigger keywords.
- `license`, `compatibility`, `metadata` — optional.

**Agent markdown frontmatter:**
- `description`, `mode` (`primary` | `subagent` | `all`), `model`, `prompt` (supports `{file:...}`), `temperature` (0.0–1.0), `top_p`, `permission`, `tools` (deprecated), `color`, `steps` (replaces deprecated `maxSteps`), `disable`, `hidden`.

**Command markdown frontmatter:**
- `description` (required), `agent`, `model`, `subtask`.
- Body uses `$ARGUMENTS` or `$1`, `$2`, …; supports `!`command`` for shell injection and `@path` for file inclusion.

### 3.4 Permission keys (v1.18.1)
`read`, `edit`, `glob`, `grep`, `list`, `bash`, `task`, `external_directory`, `todowrite`, `webfetch`, `websearch`, `lsp`, `skill`, `question`, `doom_loop`.

**Defaults (no config = permissive):**
- Most: `allow`
- `doom_loop`, `external_directory`: `ask`
- `read`: `allow`, **but `*.env`, `*.env.*` are denied by default** (so this is actually safe for the repo)
- `*.env.example`: explicitly `allow` by default

### 3.5 MCP server syntax (current)
- Local: `type: "local"`, `command: string[]`, `cwd`, `environment`, `enabled`, `timeout` (ms, default 5000)
- Remote: `type: "remote"`, `url`, `headers`, `oauth`, `enabled`, `timeout`
- Disable globally with `tools: { "mcpname_*": false }` or per-agent

### 3.6 Variables
- `{env:VAR}` — env-var substitution (empty string if unset)
- `{file:path}` — file-content substitution (relative to config dir or absolute)

## 5. Deprecations and gotchas to respect in v1.18.1

| Item | Status | Action |
|---|---|---|
| `tools: { write: false }` boolean form | **Deprecated** since v1.1.1 | Use `permission: { edit: "deny" }` etc. |
| `tools: true / false` for entire agent | **Deprecated** | Use `permission: { "*": "allow" | "deny" }` |
| `theme`, `keybinds`, `tui` keys inside `opencode.json` | **Deprecated** | Move to a separate `tui.json` (auto-migrated on read). |
| `maxSteps` on agent | **Deprecated** | Use `steps`. |
| Anthropic-subscription plugins shipped in OpenCode | **Removed** as of v1.3.0 | Cannot use Claude Pro/Max via bundled plugin; use API key only. |
| `mcp` server with single-number `timeout` | Old | Current is `timeout: 5000` (ms) under each server — but you also set it via `provider.options.timeout` for the LLM request itself. Don't conflate. |
| Linux/Windows clipboard | Gotcha | On Linux, need `xclip` / `xsel` / `wl-clipboard`. Not relevant for this repo. |
| `opencode.json` should be JSON or JSONC | OK | Both supported. JSONC allows comments — useful for project-specific config. |
| `OPENCODE_CONFIG` env var | Gotcha | Overrides all config; can mask errors. Don't use without intent. |
| `OPENCODE_CONFIG_DIR` | Gotcha | Moves `.opencode/` discovery. Don't set globally. |
| Snapshots for huge repos | Gotcha | Repo here is small (~85k points in DB, few hundred files on disk). Keep snapshots `true`. |
| Sharing mode `auto` | Privacy | Default `manual` is fine. Do **not** set to `auto` for any production-bound session. |
| `.well-known/opencode` org config | Gotcha | Loaded automatically when authenticated with a supported provider. Not relevant for personal repos. |
